# MinerU 演讲与发布叙事骨架

## 这份文档做什么

它不是 PPT 成品，而是把你给的演讲 prompt 沉淀成可复用的叙事框架，方便后续：

- 做大会演讲
- 写发布稿
- 写活动预热 / 回顾稿
- 做官网专题页

## 当前最强的一条叙事主线

### 核心结论

`下一轮 Agent 的比拼，不在模型层，而在数据处理层。`

这条线很适合把 `OpenClaw`、`MinerU`、`Sciverse` 这些能力串起来。

## 推荐的 10 页叙事逻辑

1. AI 从“动口”到“动手”
2. OpenClaw 为什么爆了
3. 下一轮竞争为什么转向数据质量
4. 数据结构化能力为什么会变成关键
5. MinerU 是谁
6. MinerU 三大产品线
7. 2026 新生态亮点
8. MinerU × OpenClaw 数据闭环
9. MDIC 赛事承接
10. Call to action

## 可以稳定复用的几句金句

- `这不是升级，是范式跃迁。`
- `Agent 战争，本质是数据质量的战争。`
- `下一轮 Agent 的比拼，不在模型层，而在数据处理层。`
- `MinerU = Agent 的数据手，让 Agent 真正看得懂世界上的文档。`

## 这条叙事为什么适合 MinerU

因为它把 MinerU 从“工具”抬升成了“基础设施”：

- 不是只讲解析效果
- 而是讲 Agent 如何真正理解文档世界
- 最终落到数据结构化能力

## 可以嵌入的产品层证据

- 开源版：私有化、部署、国产算力
- 在线 API：Flash / Precision
- Skills：自然语言调用
- MCP：IDE 与客户端直连
- RAG 插件：Dify / LangChain / RagFlow / FastGPT

## 可以嵌入的模型层证据

- `MinerU2.5`：高分辨率文档解析主干
- `MinerU-Diffusion`：OCR / 解码效率创新
- `MinerU-HTML`：网页正文提取

## 可拆成哪些内容资产

- 演讲 PPT
- 微信稿
- 公众号长文
- 小红书图文
- X / Twitter thread
- 官网专题页

## 写作时要避免的问题

- 不要只讲 OpenClaw，不讲 MinerU 的独立价值
- 不要只讲 MinerU 功能，不讲它为什么会成为 Agent 时代关键基础设施
- 不要只堆功能点，缺少“为什么现在重要”的论证
