# MinerU Knowledge Base By Role

## 产品团队

产品团队最常见的目标是：判断产品边界、确认版本变化、统一页面文案和 FAQ。

建议先看：

1. [`01-overview-and-selection.md`](01-overview-and-selection.md)
2. [`10-version-map-and-changelog.md`](10-version-map-and-changelog.md)
3. [`12-saas-vs-open-source-capability-map.md`](12-saas-vs-open-source-capability-map.md)
4. [`05-source-of-truth.md`](05-source-of-truth.md)

重点回答的问题：

- MinerU 现在到底支持哪些格式
- 开源版、SaaS、API、客户端分别解决什么问题
- `3.0` 和 `3.1` 分别新增了什么
- 哪些能力可以主推，哪些能力只能弱展示或加注释

## 运营与内容团队

运营最常见的目标是：写官网稿、公众号、选题、活动页、FAQ、GEO/AEO 内容。

建议先看：

1. [`04-content-and-geo-playbook.md`](04-content-and-geo-playbook.md)
2. [`06-published-content.md`](06-published-content.md)
3. [`09-speaking-and-launch-narratives.md`](09-speaking-and-launch-narratives.md)
4. [`11-terms-and-entities.md`](11-terms-and-entities.md)
5. [`05-source-of-truth.md`](05-source-of-truth.md)

重点回答的问题：

- 对外应该叫 `MinerU` 还是 `magic-pdf`
- 现在还能不能写 `AGPL-3.0`
- 哪些是产品能力，哪些是底层模型能力
- 哪些话术适合讲“全格式解析”，哪些场合要加版本限定

建议先看：

1. [`02-deployment.md`](02-deployment.md)
2. [`03-api-and-ecosystem.md`](03-api-and-ecosystem.md)
3. [`10-version-map-and-changelog.md`](10-version-map-and-changelog.md)

重点回答的问题：

- 本地部署和在线 API 分别怎么选

- 哪些限制来自开源引擎，哪些限制来自 SaaS 产品层

开源协作者最常见的目标是：理解仓库边界、统一术语、避免把产品概念和代码概念混写。

建议先看：

4. [`05-source-of-truth.md`](05-source-of-truth.md)

重点回答的问题：

- `MinerU`、`magic-pdf`、`MinerU-Ecosystem` 分别是什么

- 哪些说明来自官方仓库，哪些来自内部培训或 SaaS 需求文档

## 一句话导航

- 你是产品：先看版本和能力边界
- 你是运营：先看口径、已发内容、术语边界
- 你是研发：先看部署、API、版本升级影响
- 你是开源协作者：先看术语关系、模型层和 source of truth
